================================================================================
                        PATRONES DE DISEÑO APLICADOS
================================================================================

1. SINGLETON PATTERN
   ├─ Conexion.java - Instancia única de conexión
   ├─ AdministradorConnection.java
   ├─ ProductorConnection.java
   ├─ AsistenteTecnicoConnection.java
   └─ PropietarioConnection.java
   └─ Beneficio: Garantiza instancia única, thread-safe

2. FACTORY PATTERN
   ├─ DBConnectionFactory.java
   └─ Beneficio: Crea conexiones según rol sin acoplamiento

3. STRATEGY PATTERN
   ├─ DBConnection interface
   ├─ Múltiples implementaciones
   └─ Beneficio: Cambia estrategia en tiempo de ejecución

4. TEMPLATE METHOD PATTERN
   ├─ DAOBase.java
   ├─ Todas las subclases DAO
   └─ Beneficio: Reutiliza algoritmo, subclases especializan

5. DATA ACCESS OBJECT (DAO) PATTERN
   ├─ 15 DAOs especializados
   ├─ Abstrae acceso a datos
   └─ Beneficio: Separa lógica de datos de negocio

6. GENERIC PROGRAMMING
   ├─ DAOBase<T>
   ├─ ResultSetMapper<T>
   └─ Beneficio: Código reutilizable para múltiples tipos

7. INHERITANCE
   ├─ Usuario (clase abstracta)
   ├─ Productor, AsistenteTecnico, Propietario
   └─ Beneficio: Jerarquía de usuarios con rol
